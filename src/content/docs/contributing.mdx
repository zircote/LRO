## How to Contribute

This paper is open-access and we welcome contributions from the research community. There are several ways to participate.

### Providing Feedback

- **GitHub Issues**. Open an [issue](https://github.com/zircote/LRO/issues) for specific technical feedback, corrections, or suggestions.
- **GitHub Discussions**. Start a [discussion](https://github.com/zircote/LRO/discussions) for broader questions, ideas, or conversation about the research direction.

### Peer Review

We are actively seeking peer review from researchers with expertise in:

- Language model architectures
- Context window management
- Tool-augmented AI systems
- Memory and retrieval systems for LLMs
- Distributed systems and protocol design

If you would like to provide a formal review, please open an issue with the label `peer-review` or reach out through GitHub Discussions.

### Academic Collaboration

We welcome collaboration on:

- Extending the LRO protocol to new domains
- Benchmarking and evaluation
- Reference implementations
- Comparative studies with alternative approaches

### Editing the Paper

The paper source is maintained as MDX files in the [GitHub repository](https://github.com/zircote/LRO). You can suggest edits directly via pull requests.

1. Fork the repository
2. Make your changes in the `src/content/docs/` directory
3. Open a pull request with a clear description of the changes

### Code of Conduct

All contributions should be respectful, constructive, and focused on advancing the research. We value diverse perspectives and rigorous technical discourse.
